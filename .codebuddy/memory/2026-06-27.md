## 第六轮性能压测优化（全部达标）
### 优化内容
1. **搜索热门关键词预热**：@PostConstruct 启动时异步缓存 8 个高频关键词（周杰伦/晴天/陈奕迅/告白气球/稻香/夜曲/七里香/热歌）到 Redis，1.5s 间隔避免击穿 API
2. **搜索线程池 4→50**：SEARCH_EXECUTOR 从 4 扩至 50，消除 50 并发下的 API 调用排队延迟
3. **缓存污染探测器修复**：SongCacheService.getSearchCache() 中单平台结果不清空缓存改为仅告警，避免每次穿透 API 4s
4. **ES 降级优化**：ESSearchService.isAvailable() 改为不可用时每 30s 才重检，省去每次 1s HEAD 超时
5. **音频降级链串行→并行**：SongPlayService.getPlayInfo() 中四级音质 HIRES→EXHIGH→HIGHER→STANDARD 从 for 循环改为 CompletableFuture 并行探测
6. **收藏重试移出 @Transactional**：FavoriteService.toggle() 重试循环从 @Transactional 方法内移至外层非事务方法，每次重试创建新事务
