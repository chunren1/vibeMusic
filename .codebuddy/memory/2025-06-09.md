# 2025-06-09

## 修复：歌词页信息不更新

**根因**: PlayerBar.vue 的 `onSongChange` 处理器将新歌加入队列后，没有将 `currentIdx` 指向新歌索引，导致 `currentSong` 被重设为旧歌信息，歌词页（LyricsView）的歌名、歌手、封面、歌词都不会更新。

**修复内容**:
1. `PlayerBar.vue` - `onSongChange` 中加入 `currentIdx.value = newIdx` 并调用 `saveQueue()`
2. `RecentView.vue` - `vibeAudioSetSrc(url)` 补全为 `vibeAudioSetSrc(url, sourceId, songName, artist, coverUrl)`
3. `PlaylistView.vue` - 同上补全参数
4. `LikesView.vue` - 同上补全参数

## 修复：下载歌曲不同步到 RustFS

**根因（3层）**:

1. **前端不走后端下载 API**：LyricsView/HomeView/MSearchView/MPlayerView 的下载逻辑优先 fetch 当前 audio blob 直接浏览器下载，完全跳过了 `POST /api/download/{sourceId}`，歌曲不会存到 RustFS。只有 `downloadViaBackend` fallback 才走后端。

2. **SecurityConfig 阻止浏览器下载**：`GET /api/download/file/**` 需要 JWT 认证，但浏览器 `<a>` 标签下载不带 Authorization 头，导致 403。

3. **播放时不缓存**：`stream` 代理远程音频时只做透传，不写入 RustFS。

**修复内容**:

### 前端（统一走后端 API 下载到 RustFS）
- `LyricsView.vue` — 删除 fetch blob 下载，改为 `apiDownload()` + 触发 `/api/download/file/` 浏览器下载
- `HomeView.vue` — 删除 fetch blob 优先路径，直接调 `downloadViaBackend()`
- `MSearchView.vue` — 同上
- `MPlayerView.vue` — 同上

### 后端
- `SecurityConfig.java` — 新增 `GET /api/download/file/**` 和 `GET /api/download/check/**` 为 permitAll
- `SongController.java` — stream 代理远程音频时增加旁路缓存：完整请求（非 Range seek）的音频数据用 ByteArrayOutputStream 收集，播放完成后异步 `CompletableFuture.runAsync()` 写入 RustFS

## 修复：网易云 VIP 歌曲只返回 30 秒试听版

**根因**: `MUSIC_U` Cookie 决定是否能获取完整版。若 Cookie 过期或非 VIP 账号，网易云 API 对 VIP 歌曲只返回 30 秒试听（`freeTrialInfo` 不为空 或 `time <= 30000`）。原代码检测到试听后仍返回试听 URL，没有跨平台降级。

**修复内容**:

### 后端 `SongService.java` — 网易云试听自动降级 QQ 音乐
- `getPlayInfo(sourceId, name, artist)` 和 `getPlayUrl(sourceId, name, artist)` 新增重载方法，支持传入歌名/歌手
- 新增 `tryQQFallback(songName, artist, neteaseId)` 方法：当网易云所有音质均为试听时，自动在 QQ 音乐搜索同名歌曲，获取完整版 URL
- 返回结果新增 `fallbackFrom: "netease-trial"` 标记，表示 URL 来自 QQ 降级

### `SongController.java`
- `play()` 接口传递 name/artist 给 `getPlayInfo()` 以支持 QQ 降级

### 前端
- `PlayerBar.vue` — 新增 `isTrialSong` ref，在 onSongChange 中读取 `isTrial` 标记，迷你播放条显示「试听」标签
- `HomeView.vue` / `MSearchView.vue` — `song-change` 事件携带 `isTrial`/`platform`/`fallbackFrom` 信息

## 新增：双向平台降级（网易云↔QQ音乐）

原逻辑只有「网易云试听→QQ降级」。新增反向降级「QQ无URL→网易云降级」：

### `SongService.java`
- `getPlayInfo()` QQ分支：QQ无播放URL时，调 `tryNeteaseFallback(songName, artist, qqSourceId)` 搜索网易云同名歌
- `getPlayUrl()` QQ分支：同上
- 新增 `tryNeteaseFallback()` 方法：搜索网易云同名歌 → 遍历 exhigh/higher/standard → 跳过试听版 → 返回完整版URL
- 返回标记 `fallbackFrom: "qq-no-url"`

## 修复：非首页路由重新打开浏览器黑屏

**根因**: 用户在非首页路由（如 `/likes`、`/recent`）关闭浏览器后重新打开，出现黑屏。原因有二：
1. **路由守卫用 `next(false)` 阻止导航** — 当访问需登录的页面但未登录时，`next(false)` 直接中止导航，`RouterView` 无组件渲染，只剩 `#0a0a0a` 深色背景
2. **LoginModal 只在 HomeView 中** — 即使 `authStore.showLoginModal=true`，因未渲染到首页组件，登录弹窗也不会出现

**修复内容**:
- `App.vue` — 将 `LoginModal` 从 HomeView 提升到 App.vue 全局渲染（组件用 Teleport to="body"，不受路由影响）
- `router/index.js` — 路由守卫 `next(false)` 改为重定向首页：桌面端 `next('/')`，移动端 `next('/m')`，确保始终有活跃路由可渲染
- `HomeView.vue` — 移除重复的 `LoginModal` 组件（已在 App.vue 全局挂载）

## 完善：播放模式控制

原来3种模式（sequential/random/single）逻辑不完整且图标不区分，改为4种标准模式：

| 模式 | 值 | 逻辑 | 图标 |
|------|---|------|------|
| 列表循环 | `list-loop` | 队列循环播放（默认） | 双箭头循环 |
| 单曲循环 | `single` | 重播当前歌曲 | 循环箭头+居中"1" |
| 随机播放 | `shuffle` | 随机选下一首 | 交叉箭头 |
| 顺序播放 | `sequential` | 播完最后一首停止 | 单向箭头 |

**修复内容**:
- `PlayerBar.vue` — 重写4种模式切换逻辑 + onEnded/prev/next 按模式行为不同 + 单曲循环"1"用 dominant-baseline:central 居中
- `MPlayerView.vue` — prevSong/nextSong 使用 playMode 逻辑 + 监听 play-mode-change 事件同步
- `LyricsView.vue` — 同步4种模式 + 图标更新 + 事件监听
- 三端通过 play-mode-change CustomEvent 双向同步模式状态
