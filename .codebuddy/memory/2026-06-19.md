## 推荐歌单"更多"不刷新修复
- **根因**：后端 `PlaylistController.recommend()` 固定调用 `personalizedPlaylists(6)`，网易云 API 对同一 Cookie 返回固定推荐列表，每次都是同样的 6 个歌单
- **修复**：改为 `personalizedPlaylists(30)`，获取 30 个歌单后用 `Collections.shuffle()` 随机打乱取前 6 个，每次请求返回不同子集
- **影响文件**：`PlaylistController.java`（Java 后端）
- **⚠️ 需重启后端**（Spring Boot）

## crypto.randomUUID 手机端兼容性修复
- **问题**：`request.js` 中 `crypto.randomUUID()` 在部分手机浏览器不支持（旧 iOS Safari、WebView、非 HTTPS 环境等），导致 POST/PUT/DELETE 请求报错，登录失败
- **修复**：添加 `generateUUID()` 降级函数，三级兼容：`crypto.randomUUID()` → `crypto.getRandomValues()` 手动构造 UUID v4 → `Math.random()` 最终兜底
- **影响文件**：`vibemusic-web/src/api/request.js`（前端）
- **无需重启**

## 手机端用户创建歌单打不开修复
- **根因**（4个问题连环导致）：
  1. 移动端缺少 `MPlaylistView.vue`（用户歌单详情视图）
  2. 移动端路由缺少 `playlist/:id`（不含 source 参数）
  3. `MPlaylistsView.vue` 跳转使用 `name: 'playlist'`（桌面端路由名），被 `beforeEach` 守卫拦截
  4. `beforeEach` 守卫把所有 `/playlist/` 路径都拼成 `/m/playlist/:source/:id`，用户歌单没有 source 参数 → `undefined`
- **修复**：
  1. 创建 `MPlaylistView.vue`，参考桌面端 PlaylistView 逻辑 + 移动端深色 UI
  2. 添加移动端路由 `playlist/:id`（放在 `playlist/:source/:id` 之前避免路由冲突）
  3. `MPlaylistsView.vue` 跳转改为 `name: 'm-playlist'`
  4. `beforeEach` 守卫区分：`to.name === 'playlist'` → `/m/playlist/:id`，否则 → `/m/playlist/:source/:id`
- **影响文件**：`router/index.js`、`MPlaylistsView.vue`、新建 `MPlaylistView.vue`
- **无需重启**
