## CORS 零数据自动切换程序化模式
- **现象**: Web Audio 连接成功但网易云音乐资源有 CORS 限制，`MediaElementAudioSource` 数据全为零，频谱只显示一条直线
- **修复**: 
  - 新增 `corsFallback` 标志和 `_corsCheckFrames` 计数器
  - `drawSpectrum()` 中检测频率数据总和，连续10帧为零则自动调用 `switchToProgrammatic()` 切换到程序化模式
  - `switchToProgrammatic()` 关闭 AudioContext、设置 corsFallback=true、重置 analyserNode、记录 specStartTime
  - `stopSpectrum()` 重置 corsFallback 和 _corsCheckFrames

## 频谱重构: 移除 createMediaElementSource + 底部生长样式
- **播放失败原因**: `createMediaElementSource()` 对同一个 `<audio>` 只能调用一次。第二次打开歌词页时创建新 AudioContext 并再次调用会抛出 `already connected` 错误，导致音频输出断开
- **修复**: 完全移除 Web Audio / createMediaElementSource 路径，始终使用程序化模式
  - 程序化模式基于播放时间 + 正弦波 + 节拍脉冲，模拟三频段（低/中/高）韵律
  - 低频频段模拟底鼓节拍，中频模拟人声/乐器，高频模拟镲片闪烁
  - 暂停时数据自动衰减到 12%
- **样式改进**: 频谱条从底部向上生长 (`y = h - bh`)，渐变从顶部高亮绿到底部透明，辉光效果
- 简化 stopSpectrum: 只需取消 raf + 重置 canvasReady
