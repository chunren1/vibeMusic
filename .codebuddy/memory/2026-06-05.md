## 修复：搜索"晴天"没有QQ音乐的周杰伦歌曲（Redis缓存问题）
- **问题根因**: `SongService.search()` 调用的是 `neteaseApiService.search()` → `/cloudsearch` (旧单平台端点)，而不是新的 `aggregatedSearch()` → `/search` (多平台聚合端点)
- **修复内容**:
  1. `SongService.search()` 改为调用 `neteaseApiService.aggregatedSearch(keyword, 1, 20)`
  2. 解析响应格式从 `{result: {songs: [...]}}` 改为 `{data: {list: [...]}}`
  3. 使用 `parseAggregatedToDTO()` 替代 `parseToDTO()` 解析多平台统一格式
  4. `SongCacheService` 缓存键前缀从 `song:search:` 升级为 `song:search:v2:` 以自动失效旧的单平台缓存
- **修改文件**: `SongService.java`, `SongCacheService.java`
- **工具 bug**: `replace_in_file` 和 `write_to_file` 对 Java 文件全部静默失败，最终用 PowerShell `WriteAllText` 才成功写入
- **提醒**: 需重启后端服务使更改生效（`@PostConstruct` 会清除旧缓存键）

## 修复：封面和时长显示异常
- **时长问题**: `parseAggregatedToDTO()` 中 duration 是毫秒但直接赋值给了秒字段 → 添加 `/ 1000` 转换
- **封面问题**: QQ音乐 `searchQQ()` 中 `s.albumcover` 可能为空 → 新增 `albummid` 回退构建完整 URL: `https://y.gtimg.cn/music/photo_new/T002R300x300M000{albummid}.jpg`
- **修改文件**: `SongService.java` (line 149), `musicapi/server.js` (searchQQ 函数)
- **提醒**: 需重启后端 Spring Boot **和** musicapi (端口3000) Node.js 服务，浏览器需要 **Ctrl+Shift+R 硬刷新**

## 修复：底部播放栏无封面
- **根因**: `addToQueue()` 当歌曲已存在队列时不更新条目属性，旧条目（localStorage 恢复或旧版本创建）缺少 `coverUrl` → PlayerBar 读取到空 `coverUrl`
- **修复内容**:
  1. `addToQueue()` 对已有条目使用 `Object.assign()` 合并新属性（补充封面、时长等）
  2. `onSongChange()` 添加修复逻辑：若队列中已有歌曲且缺封面，补充之
  3. 添加 `console.log('[PlayerBar] song-change cover:', cover)` 调试日志
- **修改文件**: `PlayerBar.vue`
- **建议**: 用户在浏览器 Console 执行 `localStorage.clear()` 清除旧队列数据
