## 网易云封面消失修复 (01:17)
**根因**: MSearchView / MRecentView / MLikesView 的 CSS 使用 `background` 简写属性（如 `background: rgba(...) center/cover no-repeat`），会隐式设置 `background-image: none`，与 vLazyImg 指令设置的 `background-image` 内联样式产生冲突。MHomeView 用了独立的 `background-size/position/repeat` 不受影响。

**修复**:
- 3 个视图 CSS: `background` 简写 → 拆为 `background-color` + `background-position` + `background-size` + `background-repeat`
- vLazyImg 指令: 改用 `setProperty('background-image', url, 'important')` 确保覆盖；新增 `updated` 钩子处理 Vue 重渲染；`WeakSet` 追踪已应用元素避免重复

## 网易云封面消失 — 缓存污染链 (17:10)
**发现问题**: 移动端没有网易云封面但桌面端有，且渲染代码完全一致。

**真正根因**: Redis 搜索缓存污染链
1. ECONNRESET 期间 `search("热歌")` 网易云空 + QQ正常 → 合并只有QQ
2. 缓存为 `song:search:v3:热歌:all` TTL=1小时
3. 恢复后 recommendStore → buildGuest() → search("热歌") → 命中旧缓存 → 还是只有QQ
4. 桌面端 deviceId 不同，缓存是恢复后才建的（有网易云），所以正常

**修复(17:10)**:
- SongCacheService: 新增 `TTL_PARTIAL=30秒`，某平台空结果时用短TTL
- SongService.search(): 检测 `neteaseSongs.isEmpty() || qqSongs.isEmpty()` → 传 incomplete=true
- 缓存前缀 v3 → v4 清所有旧缓存
- 原理: 某平台空时只缓存30秒，恢复后30秒内自动重试，拿到完整结果

**⚠️ 后端必须重启** (SongCacheService + SongService 改动)

## 网易云封面 HTTP→HTTPS 修复 (17:41)
**真正根因**: 移动端走 cpolar HTTPS 隧道，但网易云返回的封面 URL 是 `http://p3.music.126.net/...`。HTTPS 页面加载 HTTP 资源被浏览器**混合内容策略拦截**，桌面端走 `http://localhost` 不受影响。

**测试验证**: NeteaseCloudMusicApi 直接调用正常(cloudsearch 200), `al.picUrl` 返回 HTTP 地址。
**修复**: searchNetease 封面 URL 统一 `http://` → `https://` 升级，网易云 CDN 支持 HTTPS。
**⚠️ 只需重启 musicapi** (node server.js)
