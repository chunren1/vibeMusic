### 配置类创建
- pom.xml 补充了 jjwt (0.12.6)、knife4j (4.5.0) 依赖
- 创建 application.yml：MySQL (vibe_music)、JPA update、JWT 密钥、Knife4j 文档
- SecurityConfig：JWT 无状态认证，路径 /api/auth/** 公开
- CorsConfig：允许 localhost 任意端口跨域
- JacksonConfig：统一日期格式 yyyy-MM-dd HH:mm:ss，Long→String
- JpaConfig：启用审计 @CreatedDate/@LastModifiedDate
- OpenApiConfig：接口文档访问 http://localhost:8080/doc.html
- JwtUtils：Token 生成/解析/校验（jjwt）
- JwtAuthenticationFilter：拦截 Authorization Header 提取 JWT
- Result<T>：统一 API 响应 {code, message, data}
- BusinessException + GlobalExceptionHandler：统一异常处理

### 搜索架构重构 (Redis接管缓存)
- 搜索数据从 DB song 表迁移到 Redis，DB song 表仅存储已下载歌曲
- 新增 SongDTO (dto/SongDTO.java)，轻量 POJO，字段：sourceId/name/artist/album/coverUrl/duration
- 新增 SongCacheService → Redis Key: song:search:{keyword}, TTL 1h, 启动时 @PostConstruct 自动清空旧缓存
- 修复: RestTemplate 二次编码导致中文关键词返回乱码 → buildUri() 返回 URI 对象避免重复编码
- SongService.search(): Redis缓存命中直接返回 → 未命中调网易云 cloudsearch → 缓存Redis
- PlayHistory 实体重构: song_id → sourceId+songName+artist，不依赖 song 表
- SongController.play() 改为 GET /api/songs/play?sourceId=xxx&name=xxx（不依赖DB id）
- DownloadService 重构：下载时传完整歌曲信息(name/artist/album/coverUrl/duration)并存入DB
- 前端适配: playSong(id,sourceId)→playSong(sourceId,name,artist)，download 传完整 song 对象
- 数据库: play_history 表重建，song 表清空
