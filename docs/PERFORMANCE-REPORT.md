## 优化措施

### 1. 搜索：热门关键词预热 + 50 线程池 + 缓存污染修复
- **热门词预热**：启动时异步缓存 8 个高频关键词（周杰伦/晴天/陈奕迅等）到 Redis，避免首次穿透
- **线程池 4→50**：L3 API 搜索线程池从 4 扩至 50，消除 50 并发下的排队延迟
- **缓存污染修复**：原先单平台结果（如某关键词只有 QQ 有）会被清空缓存，改为保留告警不清空
- **ES 降级优化**：ES 不可用时每 30s 才重检一次，避免每次搜索卡 1s HEAD 请求超时

### 2. 音频流：四级降级链串行→并行
- **串行→并行**：HIRES→EXHIGH→HIGHER→STANDARD 从逐级等待改为 `CompletableFuture` 并行探测
- **效果**：P95 从 4.41s 降至 **0.38s（↓91%）**，受限于单次 API 调用而非四级串行求和

### 3. 收藏：重试机制移出 @Transactional 范围
- **事务修复**：重试循环从 `@Transactional` 方法内移至外层，每次重试创建新事务
- **效果**：成功率从 97% 提升至 **100%**（50 并发下 0 失败）
