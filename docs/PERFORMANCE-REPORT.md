## 瓶颈分析

### 1. 搜索延迟（缓存命中 vs API 穿透）
- **缓存命中（median=7ms）**：极快，Redis/ES 缓存起了关键作用
- **API 穿透（p95=3.48s）**：搜索词未缓存时需穿透到 musicapi BFF 网关 → 网易云/QQ 音乐第三方 API，叠加双源并行搜索（3s 超时）和排序去重耗时
- **优化方向**：热门搜索词预热入 Redis 缓存，减少首次穿透概率

### 2. 音频流延迟（第三方依赖）
- **median=1.33s**：六级音质降级链（LOCAL→HIRES→...→FALLBACK）逐级探测导致叠加延迟
- **p95=4.41s**：部分音质无法访问时需遍历降级链
- **优化方向**：将降级链由串行改为并行（CompletableFuture.anyOf）

### 3. 收藏成功率 97%（17 次失败）
- 失败可能由于并发场景下的幂等性冲突或 DB 唯一索引竞态
- **优化方向**：增加乐观锁重试机制

## 改进后的预期

| 优化项 | 预期效果 |

| 幂等锁重试 | 收藏成功率 > 99.9% |
