# vibeMusic 前端 UI 优化计划书

> 基于 Spotify / Apple Music / 网易云 / QQ 音乐 2024 版对比分析
> 当前状态：移动端接近现代，桌面端停留在 2018 风格

---

## 一、当前差距速览

| 维度 | 移动端 | 桌面端 | 主流产品 |
|------|--------|--------|---------|
| 配色 | Velvet Night 暗色 ✅ | 浅灰白 #f5f5f5 ❌ | 全暗色 |
| 侧栏 | 无（底部 TabBar） | 浅灰 #e8e8e8 260px | 深色窄栏 72-220px |
| 播放器 | 全屏封面+光晕 ✅ | 白底功能条 | 封面模糊背景 |
| 搜索 | 防抖+历史+筛选 ✅ | 无防抖无历史 | 输入即建议 |
| 品牌色 | #2ee59a 翡翠绿 ✅ | #31c27c / #ec4141 混用 | 统一品牌色 |
| 设计 Token | mobile-theme.css ✅ | 硬编码 hex ❌ | 全站 Token |

---

## 二、优化路线图

### P1：设计系统统一（1-2 天）

**目标：全站同一套颜色、字体、间距**

| 任务 | 涉及文件 | 说明 |
|------|---------|------|
| 创建 `design-tokens.css` | 新建 | 合并 mobile-theme.css，桌面/移动共用 `:root` 变量 |
| 替换桌面硬编码颜色 | App.vue, HomeView, SearchView, PlayerBar, TopBar | `#e8e8e8`→`var(--bg-elevated)`, `#f5f5f5`→`var(--bg-base)` |
| 统一品牌色 | 全部 | 统一使用 `--primary: #2ee59a`，删除 `#ec4141` 红色 |
| 删除 emoji 图标 | SearchView, HomeView | `★📋👤` → SvgIcon 组件 |

**预期效果：全站看起来像同一个产品，不再「切换页面跳主题」**

---

### P2：桌面端暗色化（1-2 天）

**目标：桌面端与移动端视觉一致**

| 任务 | 当前 | 改为 |
|------|------|------|
| 侧栏 | `#e8e8e8` 260px | `var(--bg-elevated)` 220px |
| 主内容区 | `#f5f5f5` | `var(--bg-base)` #08080a |
| 导航项 | border-left active | 背景高亮 + 文字变色 |
| Logo 区 | 1077KB 大图+留白 | 缩小 + 下方加快捷入口 |
| 播放条 PlayerBar | 白底 #fff | 深色毛玻璃 + 封面 64px |
| 滚动条 | 浅色 thumb | 深色 `rgba(255,255,255,.1)` |

**预期效果：接近 Spotify Desktop / 网易云 PC 版的观感**

---

### P3：搜索页现代化（1 天）

**目标：搜歌体验接近网易云/QQ**

| 任务 | 说明 |
|------|------|
| 桌面搜索加防抖 | 400ms debounce |
| 搜索历史 | localStorage，含清除功能 |
| 平台筛选 Tab | netease / qq / all，桌面端补齐 |
| 搜索结果骨架屏 | 替代纯文字「搜索中...」 |
| 关键词高亮 | ES 已有 `searchWithHighlight()`，前端接 `<em>` 样式 |
| 空状态优化 | 「未找到」→ 插画 +「试试热歌」按钮 |
| 平台来源 badge | 每首歌显示「网易云」或「QQ」tag |

**预期效果：搜索体验从「勉强能用」到「流畅现代」**

---

### P4：播放器沉浸化（2 天）

**目标：桌面增加全屏播放页，移动端交互增强**

#### 桌面端

| 任务 | 说明 |
|------|------|
| 全屏播放页 | 点击 PlayerBar 封面/歌名 → 独立播放视图 |
| 封面取色背景 | 用 Canvas 取封面主色 → 渐变背景（轻量级「魔法色」） |
| 封面旋转 | 播放时 slow rotate 黑胶效果 |
| 歌词同步 | 复用 LyricsView 组件 |

#### 移动端

| 任务 | 说明 |
|------|------|
| 底部播放条背景 | 封面模糊 + 主色渐变（Spotify 风格） |
| 手势 | 上滑队列 / 下滑歌词 |

**预期效果：播放器从「功能条」升级为「音乐空间」**

---

### P5：首页发现流（2 天）

**目标：首页从「竖列表」升级为「发现流」**

| 任务 | 说明 |
|------|------|
| 横向推荐模块 | `HorizontalScrollSection.vue` 通用组件，卡片横向滚动 |
| 推荐歌曲卡片化 | 2 列网格 + 大封面（封面占 60%+ 面积） |
| 歌单推荐 | 横向滚动 |
| 问候语 | 桌面端也显示 greet |

**页面结构：**
```
┌─────────────────────────────────────┐
│  问候语 + 搜索入口                    │
├─────────────────────────────────────┤
│  Banner 轮播                         │
├─────────────────────────────────────┤
│  为你推荐 →  [卡片][卡片][卡片] →    │
│  推荐歌单 →  [歌单][歌单][歌单] →    │
└─────────────────────────────────────┘
```

---

### P6：微交互与动效（1 天）

**目标：增加「现代 App 质感」**

| 场景 | 效果 |
|------|------|
| 页面切换 | Vue `<Transition>` 滑动/淡入 |
| 收藏按钮 | 点击时 `scale` + 颜色过渡 |
| 播放/暂停 | 按钮 morph 动画（▶ ↔ ❚❚） |
| 切歌 | 封面 crossfade 300ms |
| 列表加载 | 搜索/歌单/收藏统一 skeleton |
| 空状态 | 插画 + 推荐入口（非纯文字） |

## 四、建议执行顺序

```
P1 设计 Token → P2 桌面暗色 → P3 搜索现代化
     → P4 播放器 → P5 首页流 → P6 微交互
```
