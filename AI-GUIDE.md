# vibeMusic AI 使用指南

> 你不需要会写代码。说现象，AI 来修。

---

## 一、快速上手

| 工具 | 怎么启动 | 什么时候用 |
|------|---------|-----------|
| **Claude Code** | 终端输入 `claude` | 大修改、Bug 排查、写测试、Git 操作 |
| **CodeBuddy** | VS Code 右侧侧栏图标 | 小修改、问问题、看代码 |

两个都用同一个项目文件，选你顺手的就行。

---

## 二、日常用法（复制即用）

### 修 Bug

```
修复播放历史时间为空的问题
```

> AI 会先查原因 → 写测试 → 改代码 → 运行全部测试 → 汇报结果。全程自动。

### 加功能

```
帮我加一个批量删除歌单的功能，桌面端和移动端都要
```

> AI 会建后端 API + 前端页面 + 路由注册 + 测试。

### 排查问题

```
/audit-time
```

> 扫描所有数据库表的时间字段，告诉你哪些有风险。

### 规划需求（不改代码）

```
/plan 我想加一个积分兑换功能
```

> AI 输出实施计划和风险点，你确认后才动手改代码。

### 验证修改

```
跑一下测试
```

> AI 自动运行 `mvn test` + `vitest`，出错了它自己修。

### 重启提醒

修改完 Java 文件后，AI 会自动提醒你**重启后端**。注意看它的回复。

## 三、常见任务指令

| 你想做什么 | 直接说 |
|-----------|--------|
| 修复页面样式 | `XX 页面不好看，帮我改一下` |
| 加一个搜索条件 | `歌曲搜索加一个按歌手筛选` |
| 数据库出问题 | `查一下为什么XX表的数据没写入` |
| 前端页面跑不通 | `帮我看看前端为什么报错` |
| 想了解代码逻辑 | `这个搜索功能是怎么工作的` |
| 性能太慢 | `搜索太慢了，帮我看看瓶颈在哪` |
| 想删除功能 | `帮我删掉AI助手这个功能` |

**你不用告诉 AI 怎么修**，只需说清楚**现象**（看到了什么 + 希望看到什么）。剩下的 AI 自己会查代码 → 定位问题 → 改代码 → 验证。

## 四、善用 /compact

聊久了 AI 会变慢（上下文太长），说：

```
/compact
```

它会总结当前进展，清空冗余记忆，恢复速度。

## 六、注意事项

1. **改了 Java 代码 → 必须重启后端**，AI 改完会提醒你
2. **改了前端代码 → 刷新页面**即可，Vite 热更新
3. **报未知错误** → 把红色错误日志复制给 AI 就行
4. **觉得 AI 不对** → 直接说"不对，重新做"，它会换方案
5. **AI 越改越瓦** → `/compact` 清一下，或者直接关掉重开一个对话
